# Badge

The Badge component is used to highlight and display status information, notifications, or counts. It can be used standalone or to decorate other components.

## Import

```tsx
import { Badge } from '@lazy-panda-ui/lazy-panda-ui';
```

## Usage

```tsx
import React from 'react';
import { Badge } from '@lazy-panda-ui/lazy-panda-ui';

function MyComponent() {
  return (
    <Badge content={3}>
      <Icon name="notification" />
    </Badge>
  );
}
```

## Props

| Name | Type | Default | Description |
|------|------|---------|-------------|
| `content` | `string \| number \| ReactNode` | - | Content to display in badge |
| `variant` | `'dot' \| 'standard'` | `'standard'` | Visual variant of badge |
| `color` | `'primary' \| 'secondary' \| 'success' \| 'warning' \| 'error'` | `'primary'` | Color scheme |
| `max` | `number` | `99` | Maximum value to display |
| `showZero` | `boolean` | `false` | Whether to show when content is 0 |
| `overlap` | `'circular' \| 'rectangular'` | `'rectangular'` | Shape of overlap |
| `horizontal` | `'left' \| 'right'` | `'right'` | Horizontal positioning |
| `vertical` | `'top' \| 'bottom'` | `'top'` | Vertical positioning |
| `invisible` | `boolean` | `false` | Whether badge is hidden |

```tsx

  <Badge variant="dot">

  </Badge>
</Stack>
```

## Customization

### Custom Content
```tsx
<Badge
  content={
    <Icon
      name="check"
      size="xs"
      color="white"
    />
  }
>
  <Avatar />
</Badge>
```

### Custom Styles
```tsx
<Badge
  content={5}
  style={{
    backgroundColor: '#FF6B6B',
    padding: '0 8px',
    borderRadius: '16px'
  }}
>
  <Box>Custom Badge</Box>
</Badge>
```

### Dynamic Visibility
```tsx
<Badge
  content={messageCount}
  invisible={messageCount === 0}
>
  <Icon name="mail" />
</Badge>
```

## Accessibility

### ARIA Support
```tsx
<Badge
  content={4}
  aria-label="4 unread messages"
>
  <Icon name="mail" />
</Badge>
```

## Best Practices

1. Use clear and concise content
2. Choose appropriate colors for context
3. Consider maximum values for clarity
4. Use consistent positioning
5. Handle zero states appropriately
6. Maintain readability
7. Include proper ARIA attributes
