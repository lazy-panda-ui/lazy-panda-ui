# Avatar

The Avatar component is used to represent users, entities, or organizations through images, initials, or icons.

## Import

```tsx
import { Avatar } from '@lazy-panda-ui/lazy-panda-ui';
```

## Usage

```tsx
import React from 'react';
import { Avatar } from '@lazy-panda-ui/lazy-panda-ui';

function MyComponent() {
  return (
    <Avatar
      src="/path/to/image.jpg"
      alt="User avatar"
      size="md"
    />
  );
}
```

## Props

| Name | Type | Default | Description |
|------|------|---------|-------------|
| `src` | `string` | - | Image source URL |
| `alt` | `string` | - | Alternative text for image |
| `size` | `'xs' \| 'sm' \| 'md' \| 'lg' \| 'xl' \| number` | `'md'` | Size of the avatar |
| `name` | `string` | - | Name to generate initials from |
| `icon` | `ReactNode` | - | Icon to display when no image |
| `variant` | `'circle' \| 'rounded' \| 'square'` | `'circle'` | Shape of the avatar |
| `status` | `'online' \| 'offline' \| 'busy' \| 'away'` | - | Online status indicator |
| `bordered` | `boolean` | `false` | Whether to show border |
| `loading` | `boolean` | `false` | Show loading state |

## Customization

### Custom Background
```tsx
<Avatar
  name="Custom"
  style={{
    background: 'linear-gradient(45deg, #FE6B8B 30%, #FF8E53 90%)',
    color: 'white'
  }}
/>
```

### Custom Border
```tsx
<Avatar
  src="/user1.jpg"
  alt="Bordered"
  bordered
  style={{
    borderColor: '#00C853',
    borderWidth: 3
  }}
/>
```

### Custom Status Indicator
```tsx
<Avatar
  src="/user1.jpg"
  alt="Custom status"
  status="online"
  statusProps={{
    size: 12,
    color: '#00C853',
    ringColor: 'white'
  }}
/>
```

## Fallback Behavior

### Image Loading
```tsx
<Avatar
  src="/user1.jpg"
  alt="Loading"
  loading
  fallback={<Icon name="user" />}
/>
```

### Broken Image
```tsx
<Avatar
  src="/invalid-image.jpg"
  alt="Broken"
  name="Fallback Name"
  onError={(e) => {
    console.log('Image failed to load');
  }}
/>
```

## Best Practices

1. Provide meaningful alt text
2. Use appropriate image sizes
3. Handle loading states
4. Implement proper fallbacks
5. Consider mobile displays
6. Use consistent sizing
7. Optimize image assets
